# Claude Instructions for dotfiles

## Purpose
This is a personal dotfiles repository managing a consistent Emacs-centred development environment on macOS.

## Before making structural changes
Read `Change Log.org` in the repository root. It records why things are structured the way they are, what alternatives were considered, and what patterns are intentional. Do not change established patterns without a good reason, and add an entry to `Change Log.org` when you do.

## Repository structure

```
~/dotfiles/
├── Emacs.org          — literate Emacs configuration (source of truth)
├── Readme.org         — project overview
├── Change Log.org     — decision log
├── .claude/
│   └── CLAUDE.md      — this file
├── bin/               — dotfiles scripts
│   └── tangle-dotfiles.sh
└── elisp/             — local Emacs modules (version controlled)
    └── reminders.el
```

## Emacs configuration

- **Never edit `.emacs.d/init.el` directly.** It is generated by tangling `Emacs.org`.
- To apply changes: open `Emacs.org` and run `M-x org-babel-tangle` or `C-c C-v t`.
- Third-party packages are managed with `use-package`.
- All keybindings go in `Emacs.org`. Never add keybindings inside `elisp/` modules.

## Local Emacs modules (elisp/)

- Custom functions live in `~/dotfiles/elisp/`, grouped by domain (e.g. `reminders.el`).
- Each file ends with `(provide 'module-name)`.
- The `elisp/` directory is added to `load-path` in the `* Local Packages` section of `Emacs.org`.
- New modules must be `require`d in that same section and have an entry added to `Change Log.org`.
- Keybindings for functions defined in a module go in `Emacs.org`, not in the module file.

## Adding a new module
1. Create `~/dotfiles/elisp/your-module.el`
2. End the file with `(provide 'your-module)`
3. Add `(require 'your-module)` to the `* Local Packages` section in `Emacs.org`
4. Tangle `Emacs.org`
5. Add an entry to `Change Log.org`

## Shell and tools
- `reminders-cli` is installed via `brew install keith/formulae/reminders-cli`
- Emacs runs as a server (`server-start` is in `Emacs.org`)
